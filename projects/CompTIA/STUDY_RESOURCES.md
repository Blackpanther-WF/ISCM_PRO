## Study Techniques

### Active Learning
1. **Read** - Study the material
2. **Summarize** - Write it in your own words
3. **Quiz** - Test yourself
4. **Review** - Go back to weak areas

### Spaced Repetition
- Study material Day 1
- Review Day 3
- Review Day 7
- Review Day 14
- Review Day 30

### Practice Testing
- Take quizzes multiple times
- Aim for 80%+ before moving on
- Focus on wrong answers
- Understand explanations deeply

### Study Groups
- Discuss difficult topics with others
- Explain concepts to peers
- Share resources and notes
- Practice verbally

## Domain-Specific Tips

### Domain 1: Threats, Attacks, and Vulnerabilities
**Focus Areas:**
- Know common attack types and vectors
- Understand malware classifications
- Learn threat actor motivations
- Study vulnerability assessment methods

**Study Tips:**
- Create attack flowcharts
- Memorize malware characteristics
- Practice threat modeling
- Use real-world examples

### Domain 2: Architecture and Design
**Focus Areas:**
- Zero Trust principles
- Network segmentation
- Cryptography basics
- Cloud security models

**Study Tips:**
- Diagram network architectures
- Understand encryption algorithms
- Study design patterns
- Review cloud service models

### Domain 3: Implementation
**Focus Areas:**
- Access control models
- Authentication mechanisms
- Network security controls
- Application security

**Study Tips:**
- Compare access control models
- Memorize protocols (SSH, SAML, OAuth)
- Study secure coding principles
- Review configuration best practices

### Domain 4: Operations and Incident Response
**Focus Areas:**
- Incident response procedures
- Digital forensics
- Disaster recovery
- Business continuity

**Study Tips:**
- Learn IR frameworks (NIST, SANS)
- Understand evidence handling
- Study recovery objectives (RTO, RPO)
- Review monitoring tools

### Domain 5: Governance, Risk, and Compliance
**Focus Areas:**
- Risk management frameworks
- Compliance requirements (GDPR, HIPAA, PCI-DSS)
- Security policies
- Privacy regulations

**Study Tips:**
- Compare frameworks (NIST, ISO 27001)
- Understand regulatory requirements
- Learn risk assessment methods
- Study vendor management

## Practice Exam Tips

### Before Taking a Practice Exam
- [ ] Study all materials for that domain
- [ ] Take all quizzes at least once
- [ ] Score 80%+ on quizzes
- [ ] Review weak areas

### During Practice Exam
- [ ] Manage time (90 minutes for ~90 questions)
- [ ] Read carefully
- [ ] Use elimination technique
- [ ] Mark difficult questions for review
- [ ] Don't overthink

### After Practice Exam
- [ ] Review all wrong answers
- [ ] Study explanations thoroughly
- [ ] Identify weak topics
- [ ] Retake quiz for weak areas
- [ ] Track improvement

### Time Management Strategy
- **Easy questions**: 30-40 seconds each
- **Medium questions**: 1 minute each
- **Difficult questions**: 1.5-2 minutes each
- **Review time**: 5-10 minutes
